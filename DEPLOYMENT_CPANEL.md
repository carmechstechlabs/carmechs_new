# cPanel Deployment Guide for CarMechs

This guide explains how to deploy the CarMechs application to a cPanel-based hosting environment.

## Prerequisites

1.  **Node.js Support**: Ensure your cPanel hosting has the "Setup Node.js App" feature enabled.
2.  **Database**: A Supabase project (for data) and a Firebase project (for Auth).
3.  **Domain**: A registered domain or subdomain pointed to your hosting.

## Step 1: Prepare the Application

1.  **Build the Frontend**:
    Run the following command locally to generate the production build:
    ```bash
    npm run build
    ```
    This creates a `dist` folder.

2.  **Package the Files**:
    Create a ZIP file containing the following files and folders:
    - `dist/`
    - `server.ts` (or the compiled version if you prefer)
    - `package.json`
    - `package-lock.json`
    - `.env` (Make sure to include production variables)
    - `src/services/supabaseService.ts` (and other necessary server-side dependencies)

## Step 2: Upload to cPanel

1.  Log in to your cPanel dashboard.
2.  Open **File Manager**.
3.  Navigate to your application's directory (e.g., `public_html/carmechs` or a separate folder).
4.  Upload the ZIP file and extract it.

## Step 3: Setup Node.js App in cPanel

1.  In cPanel, search for **Setup Node.js App**.
2.  Click **Create Application**.
3.  **Node.js version**: Select the latest stable version (e.g., 18.x or 20.x).
4.  **Application mode**: Set to `Production`.
5.  **Application root**: The directory where you uploaded the files (e.g., `carmechs`).
6.  **Application URL**: Your domain (e.g., `carmechs.yourdomain.com`).
7.  **Application startup file**: Set this to `server.ts` (Note: cPanel usually expects a `.js` file. If your server uses `tsx` or `ts-node`, you might need to point it to a `server.js` file generated by `tsc`).
    *   *Tip*: It's often easier to compile `server.ts` to `server.js` using `npx tsc server.ts --esModuleInterop --outDir .` before uploading.

8.  **Environment variables**: Add the following:
    - `NODE_ENV=production`
    - `SUPABASE_URL=your_supabase_url`
    - `SUPABASE_ANON_KEY=your_supabase_anon_key`
    - `SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key`
    - `GEMINI_API_KEY=your_gemini_api_key`
    - `SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`, etc. (for emails)

9.  Click **Create**.

## Step 4: Install Dependencies

1.  Once the app is created, click the **Run npm install** button in the Node.js App interface.
2.  Alternatively, use the **Terminal** in cPanel:
    ```bash
    cd path/to/your/app
    npm install --production
    ```

## Step 5: Database Setup (Supabase)

1.  Log in to your **Supabase Dashboard**.
2.  Go to the **SQL Editor**.
3.  Upload or paste the contents of `supabase_migration.sql`.
4.  Run the script to create the necessary tables and enable Realtime.

## Step 6: Realtime Configuration

1.  In Supabase, go to **Database > Replication**.
2.  Ensure that the `supabase_realtime` publication includes all the tables you want to track in real-time.
3.  The application is already configured to use `socket.io` to bridge Supabase Realtime events to the frontend.

## Step 7: Final Checks

1.  Restart the Node.js application from the cPanel interface.
2.  Visit your URL to ensure the app is running.
3.  Check the browser console for any connection errors.

## PWA Support
The app is already configured as a PWA. When users visit the site on Android (Chrome) or iOS (Safari), they will be prompted to "Add to Home Screen".
